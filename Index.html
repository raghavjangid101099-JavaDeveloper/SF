Introduction to Salesforce Admin

5 min read Topic 1 of 9

Salesforce Administrator is a key role responsible for configuring and maintaining the Salesforce platform for an organisation. As an admin, you manage users, data, security, and automation without writing code.

This section covers everything you need to pass the Salesforce Administrator certification and manage a real-world org effectively.

What you will learn

This section covers user management, security settings, automation with Flows, reporting, and data management — the core areas for every Salesforce admin.

By the end of this section you will be confident navigating Setup, creating users, assigning profiles and permission sets, and building basic automation.

Release Notes

What's new and changing in upcoming Salesforce versions — simplified.

Dynamic Forms now available on all standard objects
Admin
Dynamic Forms allow admins to place individual fields anywhere on a Lightning page, control visibility with rules, and remove the standard record detail component entirely.
Spring '25 · Generally Available
Apex null-safe operator support
Developer
Apex now supports the null-safe operator (?.) to safely navigate object chains without throwing null pointer exceptions, simplifying complex conditional logic.
Spring '25 · Beta
Flow improvements — new collection operators
Admin + Dev
Flow Builder now supports collection filter and sort elements natively, reducing the need for loops and decision elements to process record collections.
Summer '25 · Preview
LWC TypeScript support in scratch orgs
Developer
Lightning Web Components now support TypeScript in developer scratch orgs, enabling stronger type checking and improved IDE tooling for component development.
Summer '25 · Pilot
